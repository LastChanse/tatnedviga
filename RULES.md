# Правила работы

## 1. Работа с ветками
- Ветка `main` — только для релизного кода.
- Ветка `develop`— центральная ветка разработки 
- Каждая задача выполняется в ветке: `feature/имя-задачи`.
  Каждый баг фиксится в ветке `fix/имя-бага`

## 2. Процесс разработки (Git Flow)
1. Разработчик берет задачу из колонки **Todo**, переносит в **In-Progress**.
2. Создает ветку, пишет код и **Unit-тесты**.
3. Мерджит в ветку `develop`

## 3. Стиль кода
1. Используем стандарт: PEP8 для `python`, Google JavaScript Guide для `javascript`.
2. Все методы должны быть описаны комментариями: что делает метод, описание входных параметров, выходное значение, side-effects.
3. Добавлять пояснения к несамоописывающемуся коду.
4. Не должно быть непонятных "магических" констант.
